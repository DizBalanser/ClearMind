\section{Recommended Screenshot Order}

The fastest capture order is:

\begin{enumerate}
  \item Start backend and frontend.
  \item Log in with a demonstration user.
  \item Seed or create several tasks, ideas, context facts, and graph relationships.
  \item Capture dashboard, chat, database, graph, schedule, and profile pages.
  \item Open generated API documentation and capture the overview page.
  \item Run or open coverage evidence and capture the result.
  \item Open CI evidence and capture the passing pipeline.
  \item Record the final walkthrough and capture one representative frame.
\end{enumerate}

\section{Image Quality Rules}

Each screenshot should be readable in print. Prefer light mode if dark-mode contrast is weak on paper. Crop only irrelevant browser chrome, not important application state. Keep filenames exactly as specified so the LaTeX source can discover the images without edits. If an image contains sensitive content, replace the data with demonstration values before capturing it.
